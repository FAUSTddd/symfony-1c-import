# Руководство по вкладу в проект

Спасибо за ваш интерес к улучшению проекта! Этот документ описывает процесс вклада в проект.

## Как мы работаем

Мы открытый к вкладу проект, и мы приветствуем вклад от сообщества. Мы используем GitHub для отслеживания багов и запросов функций, а также для управления вкладами.

## Как начать

1. Найдите проблему, которую вы хотите решить, или предложите новую функцию
2. Обсудите предложение в разделе issues, если это необходимо
3. Сделайте fork репозитория
4. Создайте ветку для вашей работы
5. Внесите изменения и отправьте pull request

## Структура репозитория

- `src/` — исходный код библиотеки
- `tests/` — тесты
- `config/` — конфигурационные файлы

## Руководства по коду

- Следуйте PSR-12 для PHP кода
- Используйте PHP 8.2+ синтаксис
- Пишите тесты для нового кода
- Убедитесь, что все тесты проходят перед отправкой PR

## Процесс отправки pull request

1. Сделайте fork репозитория
2. Создайте ветку (`git checkout -b feature/AmazingFeature`)
3. Сделайте коммит (`git commit -m 'Add some AmazingFeature'`)
4. Отправьте ветку (`git push origin feature/AmazingFeature`)
5. Откройте Pull Request

## Именование веток

- `feature/` — для новых функций
- `fix/` — для исправлений багов
- `docs/` — для изменений в документации
- `refactor/` — для рефакторинга кода

## Сообщение о багах

При создании issue с багом, пожалуйста, включите:

- Описание проблемы
- Шаги для воспроизведения
- Ожидаемое поведение
- Фактическое поведение
- Версии PHP и Symfony
- Любые другие релевантные детали

## Запросы функций

При создании issue для запроса функции, пожалуйста, включите:

- Описание запрашиваемой функции
- Примеры использования
- Причины, почему это полезно для сообщества

## Вопросы?

Если у вас есть вопросы, вы можете открыть issue с меткой `question`.
